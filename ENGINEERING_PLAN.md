# 动画教学工程规划

## 总目标

- 把当前工程从“少量模块的演示页面”升级为“可持续扩展的儿童互动教学平台”。
- 在保持游戏性、趣味性、互动性的同时，让界面更加现代、简洁、移动端友好。
- 让后续多个智能体可以并行开发不同模块，同时保持统一的教学结构、导航方式和页面体验。

## 当前已落地的工程能力

### 1. 应用壳层
- `src/App.tsx` 已收敛为基于模块注册表的 `AppShell`
- 顶部课程导航与底部进度条由统一页面序列驱动
- 首页与课程页视觉结构分离，便于继续扩展学习档案、推荐下一课等能力

### 2. 模块组织
- 现有课程已拆分到 `src/modules/*`
- `src/components/animations.tsx` 承担共享动画与代码展示能力
- `src/modules/chicken-rabbit/index.tsx` 已提供独立模块入口，供壳层统一接线

### 3. 课程注册表
- `src/course/registry.ts` 统一维护：
  - 模块标题与简介
  - 首页入口
  - 页面顺序
  - 顶部导航与底部进度条标签
- 首页不再手工硬编码所有课程卡片

### 4. 界面与移动端体验
- 首页升级为更现代的课程导航面板
- 顶部使用粘性课程导航，底部保留可点击的进度条
- 全局基础样式已优化为更适合手机竖屏浏览的间距、圆角与层次

## 下一阶段目标

### 1. 内容层
- 把练习题、案例、动画步骤继续从页面组件中抽离出来
- 逐步建立统一的数据 schema，减少页面内部硬编码

### 2. 共享能力层
- 继续沉淀共享图示组件，如网格图、排序柱状图、练习反馈卡、奖励卡片
- 统一按钮、卡片、题目反馈、奖励提示等交互模式

### 3. 激励层
- 增加星星积分、徽章、连续学习奖励
- 建立模块完成记录和学习解锁逻辑

### 4. 学习路径层
- 增加推荐下一课
- 增加学习地图与学习档案
- 增加主题筛选、难度标记和家长查看页

## 后续迁移顺序建议

### 第 2 阶段：共享组件继续下沉

### 第 3 阶段：样式体系继续拆分

### 第 4 阶段：长期能力接入

- 徽章与积分
- 学习地图

## 多智能体协作规则

### 模块智能体
- 负责具体知识模块的页面、案例、练习和动画数据

### 基础设施智能体
- 负责组件库、动画引擎、模块注册表、样式体系

### 文档智能体
- 负责 README、AGENTS、TODO 路线图维护

### 质量智能体
- 负责 lint、build、页面冒烟测试、GitHub Pages 兼容检查

## 必须长期坚持的约束

- 新模块必须接入首页导航
- 新模块必须遵循统一教学页面结构
- 新模块完成后必须更新 README
- 新模块完成后必须更新 TODO 路线图状态
- 改动部署相关内容时必须确认 GitHub Pages 兼容
- 开发完成后必须先校验，再 commit 与 push
