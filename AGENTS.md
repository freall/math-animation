# Math Animation Agents Guide

## 项目目标

- 本项目是面向儿童的交互式教学网页。
- 当前包含三个学习主题：容斥原理、最短路线标数法、C++ 循环入门。
- 所有新增内容都要保持“图形化 + 动画化 + 例题 + 练习”的教学结构。

## 开发原则

- 优先复用现有的 React 单文件模式，主要逻辑集中在 `src/App.tsx`。
- 优先复用 `src/App.css` 里的页面卡片、按钮、动画和练习样式。
- 新模块必须接入首页导航，并加入底部进度条页面序列。
- 面向小朋友表达：标题清晰、语句短、案例具体、反馈正向。
- 图形和动画要服务讲解，不做复杂但难懂的交互。

## 功能交付要求

- 每个教学模块至少包含：
- 概念引入页
- 基本语法或核心规则页
- 动画演示页
- 实际案例页
- 练习页
- 新增案例时，尽量使用贴近儿童生活的情境，如星星、花盆、跳绳、积分、路线等。

## 工程规则

- 开发完成后必须先验证，再提交代码。
- 默认验证命令：
- `pnpm lint`
- `pnpm build`
- 若改动影响 GitHub Pages，额外确认构建产物路径与 Pages 配置一致。
- GitHub Pages 需要同时兼容：
- `https://freall.github.io/math-animation/`
- `https://m.aipanda.cc/`
- 资源路径必须兼容子路径部署和自定义域名部署。

## 文档规则

- 每次新增模块、调整部署、修改开发流程后，要同步更新 `README.md`。
- `README.md` 视为持续维护文档，不允许长期落后于实际代码。
- 本文件 `agents.md` 视为多代理协作规则，后续开发前应先阅读。

## Git 流程

- 用户要求：每次开发完成并验证通过后，自动执行 commit 并 push 到远程。
- commit 信息要简洁明确，优先使用 `feat:`、`fix:`、`docs:` 等前缀。

## 当前部署事实

- GitHub Pages 使用 Actions 工作流发布 `dist`。
- 不能重新引入 Jekyll Pages 工作流，否则可能覆盖 Vite 产物。
